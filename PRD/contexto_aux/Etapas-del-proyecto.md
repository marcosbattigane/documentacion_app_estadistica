## Fase 1 – Definición del alcance y objetivos del sistema

La primera fase consiste en definir con claridad el alcance del proyecto y los objetivos que tendrá el sistema en su versión inicial. En esta etapa se debe establecer qué problemas busca resolver la aplicación y cuál será su público objetivo, en este caso entrenadores de básquet que necesiten registrar y analizar estadísticas de partidos y competiciones. También se debe determinar cuáles serán las funcionalidades mínimas necesarias para una primera versión funcional del sistema, evitando incorporar características demasiado complejas desde el inicio. Esta fase incluye la definición del flujo general de uso de la aplicación, identificando cómo un entrenador creará su cuenta, registrará su club, gestionará categorías, competencias, equipos y jugadores, y cómo podrá cargar estadísticas de partidos. El resultado de esta fase debe ser un documento claro que describa las funcionalidades principales del sistema y los límites del proyecto para su primera versión.

## Fase 2 – Modelado conceptual del sistema

Una vez definido el alcance, la segunda fase consiste en modelar conceptualmente la estructura del sistema. En esta etapa se identifican todas las entidades que formarán parte del software y las relaciones entre ellas. Algunas de las entidades principales serán usuarios, clubes, categorías, competencias, equipos, jugadores, partidos y estadísticas. El objetivo de esta fase es comprender cómo se relacionan los datos dentro del sistema y cómo fluirá la información desde que un partido se registra hasta que se generan estadísticas acumuladas. También se debe definir qué información almacenará cada entidad y qué operaciones se podrán realizar sobre ellas. El resultado de esta fase suele representarse mediante diagramas conceptuales o diagramas entidad-relación que describen la estructura lógica del sistema antes de implementar la base de datos.

## Fase 3 – Diseño de la base de datos

En esta fase se realiza el diseño completo de la base de datos que almacenará toda la información del sistema. A partir del modelo conceptual definido previamente se construye la estructura de tablas, definiendo campos, claves primarias y relaciones entre tablas. El objetivo es diseñar una base de datos coherente, normalizada y preparada para almacenar información de múltiples clubes, competiciones, equipos y jugadores. También se debe contemplar cómo se almacenarán las estadísticas de cada jugador en cada partido, de forma que luego puedan calcularse estadísticas acumuladas o avanzadas. Esta fase es crítica porque la estructura de datos condiciona gran parte del desarrollo posterior del sistema. Al finalizar, debe existir un diseño claro y documentado de la base de datos que será utilizado por toda la aplicación.

## Fase 4 – Desarrollo de la capa de gestión de datos

Una vez definida la base de datos, la siguiente fase consiste en desarrollar la capa del sistema encargada de gestionar el acceso y la manipulación de los datos. Esta capa será responsable de todas las operaciones relacionadas con el almacenamiento y la recuperación de información, como crear clubes, registrar jugadores, guardar partidos o consultar estadísticas. El objetivo es centralizar la interacción con la base de datos en un conjunto organizado de funciones o servicios que permitan mantener el código ordenado y desacoplado de la interfaz gráfica. En esta etapa se valida que el modelo de datos diseñado en la fase anterior funciona correctamente en la práctica y permite realizar todas las operaciones necesarias para el funcionamiento del sistema.

## Fase 5 – Implementación del sistema de importación de datos externos

Una de las funcionalidades clave del proyecto es la posibilidad de importar planillas de estadísticas generadas por herramientas externas, como las planillas de la aplicación Ges Deportivo. En esta fase se desarrolla el módulo encargado de leer estos archivos, interpretar su estructura y transformar los datos en información compatible con el modelo de la base de datos del sistema. El objetivo es automatizar la carga de estadísticas de partidos para evitar que el entrenador tenga que ingresar todos los datos manualmente. Esta etapa incluye el análisis del formato de los archivos, la extracción de la información relevante y su posterior almacenamiento en la base de datos. También es importante validar los datos importados para asegurar su consistencia.

## Fase 6 – Desarrollo del motor de estadísticas y análisis

En esta fase se implementa la lógica encargada de procesar la información almacenada en la base de datos para generar estadísticas útiles para el entrenador. Esto incluye el cálculo de estadísticas acumuladas por jugador, por equipo y por competencia, así como la generación de métricas derivadas a partir de las estadísticas básicas. El objetivo es transformar los datos registrados en información que permita analizar el rendimiento de jugadores y equipos. También se pueden definir las primeras estadísticas avanzadas que aporten valor al análisis deportivo. Esta capa del sistema será la responsable de generar los datos que posteriormente serán visualizados en la aplicación.

## Fase 7 – Desarrollo de la interfaz de usuario

Una vez que la base de datos, la gestión de datos y el motor estadístico se encuentran funcionando correctamente, se inicia el desarrollo de la interfaz de usuario de la aplicación de escritorio. Esta interfaz permitirá a los entrenadores interactuar con el sistema de manera intuitiva, facilitando la creación de clubes, categorías, equipos y jugadores, así como la carga de partidos y la visualización de estadísticas. En esta fase se diseñan las distintas pantallas del sistema, se define la navegación entre secciones y se implementan los formularios necesarios para ingresar información. El objetivo es construir una experiencia de uso clara y sencilla que permita acceder a todas las funcionalidades del sistema sin requerir conocimientos técnicos.

## Fase 8 – Visualización y herramientas de análisis

Con la interfaz básica funcionando, la siguiente fase se enfoca en mejorar la forma en que se presentan las estadísticas al usuario. En esta etapa se incorporan herramientas de visualización que permitan analizar la información de manera más clara, como gráficos, comparaciones entre jugadores o equipos, y resúmenes estadísticos de partidos o competiciones. El objetivo es que el sistema no solo almacene estadísticas, sino que también ayude al entrenador a interpretarlas y utilizarlas para la toma de decisiones deportivas.

## Fase 9 – Pruebas, validación y optimización

Antes de considerar el proyecto finalizado, es necesario realizar una etapa de pruebas exhaustivas del sistema. Durante esta fase se verificará que todas las funcionalidades funcionen correctamente, que la importación de datos sea confiable y que las estadísticas calculadas sean precisas. También se evaluará la usabilidad de la interfaz y el rendimiento general de la aplicación. El objetivo es identificar errores, corregirlos y optimizar el funcionamiento del sistema antes de su distribución. Esta fase puede incluir pruebas con datos reales de partidos para validar el comportamiento del sistema en situaciones reales de uso.

## Fase 10 – Empaquetado y distribución de la aplicación

La fase final del proyecto consiste en preparar la aplicación para su distribución y uso por parte de los entrenadores. Esto implica generar una versión instalable del software que pueda ejecutarse fácilmente en una computadora sin necesidad de configurar el entorno de desarrollo. También se puede preparar documentación básica para el usuario, explicando cómo instalar la aplicación y cómo utilizar sus principales funcionalidades. El objetivo de esta fase es transformar el proyecto desarrollado en un producto utilizable en un entorno real.
